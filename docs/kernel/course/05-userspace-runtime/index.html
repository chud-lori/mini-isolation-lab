Userspace Runtime: Init Without Ring-3

Mini Kernel now has an actual userspace-shaped program path, but it is still simulated inside the kernel build. That distinction is important and intentionally visible.

Source focus: kernel/user.c
All lessonsOpen source fileRead lesson

Article

Read this with kernel/user.c open beside it. The goal is understanding the path, not memorizing every instruction or helper.

Before You Start

  • You understand that userspace normally means less-privileged program execution.
  • You know arrays, structs, and simple interpreter loops in C.

Terms Before Code

Userspace

Program code that should run with less privilege than the kernel.

init

The first user program in many OS designs.

Ring-3

The normal low-privilege CPU mode for user programs on x86.

Runtime

Support code that executes or hosts a program model.

What Userspace Means Here

Mini Kernel has a userspace-shaped path: a program has memory, fd slots, operations, syscall frames, and an exit status. That makes it more than a note in the docs. It can run through the syscall contract and be tested.

What Is Still Simulated

The CPU is not switching to hardware ring-3 yet. The runner interprets a built-in list of user operations and calls ksyscall_entry directly. This is honest scaffolding: the data model is useful, but privilege isolation is not complete.

The Init Program

The built-in init writes a banner, opens /etc/motd, reads bytes, writes them back out, closes the fd, and exits. That sequence exercises write, open, read, close, and exit without needing a disk driver or ELF user-program loader yet.

How This Grows

The same course path can grow toward page tables, a real trap instruction, scheduler state, and binary loading. The important design choice is that the syscall contract already has somewhere to live.

Source Reading Checklist

  • Find the init operation list.
  • Find where user memory is initialized.
  • Find where exit status is captured.

Exercises

  • Change the init message and run tests.
  • Describe which part would disappear when real user binaries are loaded.

Now vs Later

Now: kuser_op describes a built-in init operation, the runner creates a trap-shaped frame, ksyscall_entry dispatches it, and the result is copied back. Later: a real user binary runs in ring-3, a CPU trap enters the kernel, the kernel saves registers, and the same syscall contract can handle the request.

Checkpoint

Explain what is simulated today and what must change for hardware user mode.
