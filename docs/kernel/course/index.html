Mini Kernel Course

A beginner-friendly, step-by-step OS course built from Mini Kernel. Each lesson is its own article with vocabulary, plain-language explanation, source-reading guidance, and exercises.

Start HereOpen source

Who This Is For

This course assumes basic terminal use and basic C. It does not assume you already know OS development, x86 boot, syscalls, privilege rings, or filesystems. Each lesson introduces the vocabulary before asking you to read source.

How To Study It

  • Read the lesson once without code.
  • Open the source file named in the lesson.
  • Use the code walk checklist to trace only the important path.
  • Answer the checkpoint before moving forward.
  • Run the build check after finishing the sequence.

Lessons

Work through these in order: mental model, boot sector, stage2 monitor, kernel entry, syscall entry, userspace runtime, first filesystem, then build verification.

Boot Sector: The First 512 Bytes

The machine starts before there is an executable loader, process model, or C runtime. This lesson follows the BIOS contract and the exact reason the first Mini Kernel binary must fit into one sector.

Stage2 Monitor: A Visible Interface

Stage2 is still real-mode assembly, but it gives the boot image a visible monitor. It turns a raw sector load into something you can interact with and reason about.

Syscall Entry: Trap-Shaped Boundary

The syscall boundary is shaped like a CPU trap even though it is still called directly by the current userspace runner. This lets the validation logic be real and testable today.
