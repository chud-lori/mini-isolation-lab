Boot Flow

The working bootable path today follows the machine before there is a process, a shell, a filesystem, or a C runtime.

01

Firmware

BIOS reads the first disk sector into physical memory at 0x7c00 and jumps to it.

02

Boot Sector

The 512-byte loader sets real-mode state, saves the boot drive, and reads contiguous stage2 sectors.

03

Stage2

The monitor runs at 0800:0000, handles simple commands, and names the next handoff: kernel, userspace, and the init program.

04

Kernel Scaffold

The freestanding C kernel path owns serial logging, panic handling, checked helpers, file lookup, and syscall dispatch; serial and panic code compile as kernel objects, while the portable syscall, filesystem, descriptor, and userspace pieces are host-tested today.

Binary Layout

Mini Kernel uses flat binaries for the current BIOS path and keeps the planned Limine/Multiboot ELF kernel path separate.

Current Vs Planned

The docs keep these paths separate so the boot status is clear.

Works Today

make all builds the raw BIOS disk image, validates the 512-byte boot sector and sector-aligned stage2 monitor, compiles kernel objects including early GDT/IDT setup, and runs host tests for syscalls, filesystem behavior, syscall-entry frames, descriptor tables, and userspace init.

Planned Bootloader Path

boot/multiboot2.S, boot/linker.ld, and boot/limine.cfg are scaffolding for a Limine/Multiboot-style ELF kernel route. They are not the default working boot image yet.

Userspace Shape

The current userspace is a tiny interpreted init program run through kuser_run, backed by validated task memory, task-local file descriptors, and the shared ksyscall_entry frame.

Init

kernel_start launches kuser_init_program, whose ops write a banner, open /etc/motd, read it, close it, and exit.

User Memory

Every syscall checks the requested pointer range against user_base and user_len before copying or printing bytes.

Entry Contract

ksyscall_entry accepts a trap-shaped frame, calls ksyscall_dispatch, and writes retval, error, and exit state back to the frame.

Syscall Dispatcher

ksyscall_entry is the actual userspace syscall entry today. It is still called directly by the userspace runner, not by a CPU trap; a future trap handler can populate the same frame and reuse this validation path.

What To Build Next

The next steps keep the lab small while turning tested pieces into booted pieces.

Boot Handoff

Either link the C kernel into the BIOS stage2 handoff or make the Limine/Multiboot path the explicit bootloader route.

Fault Diagnostics

Add real exception handlers to the loaded IDT so early CPU faults print useful register state instead of resetting silently.

User Mode

Add a real trap entry and hardware ring-3 transition that populate the syscall frame already covered by host tests.

Program Loading

Replace the built-in simulated init program with a tiny user binary loader and a minimal task lifecycle.

Mini Kernel keeps unsafe machine boundaries visible, tests portable logic on the host, and grows userspace through the same syscall contract the future trap entry will use.
