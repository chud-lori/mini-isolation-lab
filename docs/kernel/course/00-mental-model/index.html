Before touching boot assembly or syscall code, you need the map. Mini Kernel is not a normal application. It is a small operating-system specimen where the early machine path, kernel boundary, syscall contract, and userspace model are kept visible enough to study.

A normal program starts after an operating system has already done a lot of work. The OS created a process, chose virtual memory mappings, connected file descriptors, loaded code, prepared a stack, and finally called into the program. That is why a normal C program can begin with main and immediately use familiar services. A kernel starts before those services exist. If it wants output, memory management, files, users, tasks, or safety rules, it must build them.

The first helper is firmware. In the BIOS path, firmware reads the first 512-byte disk sector into memory and jumps to it. That first sector is boot code, not the kernel proper. Its job is to do the smallest reliable handoff. Mini Kernel then uses a stage2 monitor so you can see something running before the C kernel path is fully boot-linked.

There are two important paths today. The BIOS image path runs the boot sector and stage2 monitor. The freestanding C kernel path compiles serial and panic code, then host-tests syscall dispatch, the in-memory filesystem, descriptor-table builders, and the simulated userspace init program. Keeping that distinction honest is part of the course: you should know what boots today and what is being prepared for the next handoff.

Key Ideas

Normal appLoaded by an existing OS; can assume process services.
Boot codeReceives the firmware jump; prepares the next stage with almost no help.
Kernel codeOwns the rules for memory, files, syscalls, tasks, and devices.
Read the source with one question in mind: what state enters this lesson, what code changes that state, and what state should exist afterward?

Source Reading Guide

Do not try to understand every instruction or helper in one pass. First trace the main path. Then return to the error path. Finally, connect the code back to the checkpoint.
