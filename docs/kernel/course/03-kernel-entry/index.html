Kernel Entry: Serial First

The freestanding C kernel path starts with the least glamorous but most valuable primitive: logging. Serial output gives the kernel a voice before higher-level devices exist.

Source focus: kernel/main.c
All lessonsOpen source fileRead lesson

Article

Read this with kernel/main.c open beside it.

Why Serial Comes First

A kernel cannot assume stdout, files, terminal emulators, or a process runtime. Serial logging gives early code a narrow output path that can be tested and reused while the rest of the OS grows.

The Startup Shape

kernel_start initializes logging, prepares syscall-facing task state, writes a banner through the syscall path, launches the built-in init program, checks its exit status, and halts cleanly.

Failure As A Design Tool

The entry code treats failed copies, failed writes, and bad init exits as visible kernel events. That makes the course easier to debug because incorrect boundaries fail loudly.

Checkpoint

Identify what happens if the banner copy or syscall write fails.
