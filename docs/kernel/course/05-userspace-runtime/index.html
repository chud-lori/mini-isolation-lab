Userspace Runtime: Init Without Ring-3

Mini Kernel now has an actual userspace-shaped program path, but it is still simulated inside the kernel build. That distinction is important and intentionally visible.

Source focus: kernel/user.c
All lessonsOpen source fileRead lesson

Article

Read this with kernel/user.c open beside it.

What Is Real Today

The init program has its own memory buffer, fd slots, operation list, syscall frames, and exit status. It performs write, open, read, write, close, and exit through ksyscall_entry.

What Is Simulated

The CPU is not switching to hardware ring-3 yet. The runner interprets user operations and calls the syscall entry directly. The model is useful because pointer validation, fd state, and syscall results are still exercised.

The Next Boundary

Real userspace will need page tables, privilege transitions, an interrupt/trap entry, a scheduler contract, and a way to load user programs as binaries rather than built-in operation lists.

Checkpoint

Explain what is simulated today and what must change for hardware user mode.
