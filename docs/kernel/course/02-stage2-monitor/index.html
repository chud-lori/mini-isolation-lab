Stage2 Monitor: A Visible Interface

Stage2 is still real-mode assembly, but it gives the boot image a visible monitor. It turns a raw sector load into something you can interact with and reason about.

Source focus: boot/stage2.S
All lessonsOpen source fileRead lesson

Article

Read this with boot/stage2.S open beside it. The goal is understanding the path, not memorizing every instruction or helper.

Before You Start

  • You understand Lesson 01: BIOS loads the first sector, and the first sector loads stage2.
  • You know that keyboard input arrives as bytes or scan-like values that software can interpret.

Terms Before Code

Monitor

A tiny interactive command loop, not a full shell.

Command dispatch

Comparing input text and jumping to the handler for a matching command.

Halt

Stopping the CPU until reset or interrupt, depending on context.

What Stage2 Adds

Stage2 has more room than the boot sector, so it can be educational and interactive. It prints a banner, accepts commands, and shows the user that the image is alive. It is still not a full operating system; it is a visible checkpoint between firmware and kernel design.

Input To Command

The monitor loop repeatedly accepts characters, updates a command buffer, and compares the resulting text with known commands. This is the same broad pattern as a shell, but far smaller: read input, parse command, dispatch handler, print response.

Why This Matters For Learning

A boot path can feel invisible because early code often fails before logs exist. The monitor gives learners something concrete: if help/about work, then the boot sector loaded stage2, stage2 is executing, and text I/O is functioning.

Where The Kernel Story Appears

The help/about text names the next pieces of the OS: kernel entry, syscall entry, userspace, and init. Stage2 is not pretending those are fully boot-linked yet; it is exposing the roadmap in the running image.

Source Reading Checklist

  • Find the command buffer and the command comparison code.
  • Find the help/about command handlers.
  • Find the halt or reboot paths and describe what they do.

Exercises

  • Add a harmless new monitor command that prints one line.
  • Write down the minimum state the boot sector must pass to stage2.

BIOS I/O Calls

The monitor can use BIOS keyboard and text services while it remains in real mode. Keyboard input commonly comes through int 0x16, and text output commonly uses int 0x10. These calls are early conveniences, not permanent kernel drivers.

Input Buffer Model

The monitor keeps a small command buffer. Characters are appended until Enter, then the buffer is compared against known command strings. Backspace and length limits matter because the monitor has no shell library protecting it.

Checkpoint

Trace how a key press becomes a monitor command and identify where the help/about text is stored.
